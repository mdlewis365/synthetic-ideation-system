# Workflow Example

## Scenario

A user wants to explore energy conservation for small autonomous robots.

## Input

- Exploration Vector: adaptive energy conservation for small autonomous robots
- Target Exploration Domain: battery-powered inspection and field robots
- Objective Framing: invent a mechanism that reduces energy consumption without increasing battery size
- Allowed Approaches: adaptive duty cycling, passive sensing, power-aware locomotion, low-power standby states
- Reject If: the idea depends on unrealistic battery chemistry, large external infrastructure, or continuous high-power computation

Optional controls can bias the search toward behavioral adaptation, causal mechanisms, or avoiding obvious low-power processor substitutions.

## Mode

The user selects Mechanism Discovery.

## Output

SIS returns a structured concept candidate that includes a concept name, operating principle, assumptions, risks, and validation steps.

## What Is Not Included

The example is fictional. It excludes private prompt text, raw SIS outputs, evaluator details, model-routing internals, implementation paths, logs, and user data.
