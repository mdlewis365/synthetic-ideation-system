# Output Structure

## Public-Safe Output Shape

SIS concept output can be publicly described as including:

- concept name
- problem addressed
- proposed mechanism or architecture
- operating principle
- intended use
- assumptions
- risks and unknowns
- review notes
- suggested validation steps

Exact private formatting may vary by invention mode and private prompt assembly.

## Human Review Labels

Outputs should be treated as invention candidates, not validated inventions.

## What Is Not Included

This document does not include private output templates, scoring fields, evaluator traces, prompt text, raw generated outputs, or patent strategy details.
