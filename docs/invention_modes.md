# Invention Modes

## Purpose

Invention Modes describe the kind of invention work the user wants SIS to perform. The user selects the mode.

## Modes

### Mechanism Discovery

Used when the user wants to explore a causal mechanism, operating principle, or technical effect.

### System Architecture

Used when the user wants to explore components, relationships, interfaces, dependencies, or high-level system structure.

### Process Innovation

Used when the user wants to explore a new or improved process, workflow, operation sequence, or transformation method.

### Algorithmic Method

Used when the user wants to explore a computational, mathematical, procedural, optimization, classification, or decision-making method.

### Hybrid System Development

Used when the invention direction combines multiple system types, technical domains, mechanisms, or architectural layers.

### Constraint-Inversion

Used when a limitation, bottleneck, failure condition, or constraint is treated as the driver for a new mechanism, process, architecture, or method.

## What Is Not Included

This file does not disclose mode-specific private prompts, scoring rules, evaluator internals, or governance implementation.
